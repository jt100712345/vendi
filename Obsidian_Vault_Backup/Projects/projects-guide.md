# Projects Guide

This folder contains notes and resources related to active projects or ongoing tasks.

**Guidelines:**
- **Structure:** Each project should ideally have its own sub-folder (e.g., `Project Name/`).
- **Core Files:** Within each project folder, consider having:
    - `README.md`: Project overview, goals, and current status.
    - `tasks.md`: A list of actionable tasks, perhaps with checkboxes.
    - `meeting-notes.md`: Records of discussions and decisions.
- **Naming Convention:** Use clear, descriptive project names for folders.
- **Linking:** Extensively link project notes to relevant `Thinking` (for ideas, research) and `Resources` (for reference materials).
- **Status Updates:** Regularly update the `README.md` or `tasks.md` with progress.
- **Archiving:** Once a project is complete, consider archiving it to a dedicated `Archive` sub-folder within `Projects` or a separate `Archive` top-level folder, ensuring easy future reference without cluttering active projects.
